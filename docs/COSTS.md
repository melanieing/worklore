## 1. 사용량 모델 (활성 유저 1명/월 가정)

주 2~3회 기록 = 월 ~10 entries 기준.

| 작업 | 횟수/월 | 토큰 (in/out, 회당) | 등급 |
|---|---|---|---|
| 역질문 생성 | 10 | ~800 / ~150 | light |
| 태깅/분류 | 10 | ~500 / ~50 | light |
| 성취 카드 생성/갱신 | 5 | ~2,000 / ~600 | quality |
| 이력서 불릿 | 3 | ~1,500 / ~300 | quality |
| 월간 임팩트 요약 | 1 | ~6,000 / ~800 | quality (batch 가능) |

월 합계: light ~13K in / 2K out, quality ~20.5K in / 3.8K out.

## 5. 비용 최소화 전략 (실행 순서)

1. **[M2-0] 프로바이더 추상화** — Vercel AI SDK로 LLM 호출부 교체.
   모델은 env로 주입(`AI_MODEL_LIGHT/QUALITY`), 벤더 교체가 배포 없이 가능해야 함.
   현재 `src/lib/ai/`의 Anthropic SDK 직결 코드는 M2 시작 시 이 레이어로 이관.
2. **최저가 라우팅 기본값** — light=GPT-5.4 Nano(또는 Gemini Flash-Lite), quality=Gemini 2.5 Flash.
   품질 미달 시에만 한 단계씩 상승. 상승 판단은 감이 아니라 Q-3 프롬프트 회귀 테스트로.
3. **비실시간 작업은 Batch API (50% 할인)** — 월간 요약(M4-4), 카드 일괄 재생성(M2-5).
4. **프롬프트 캐싱** — 시스템 프롬프트를 안정된 prefix로 유지(타임스탬프 등 삽입 금지).
5. **호출 자체를 줄이는 제품 설계**
   - 역질문은 entry당 1회 호출로 1~3개 일괄 생성 (이미 그렇게 구현됨)
   - 카드 자동 생성 금지 → 답변 완료 후 사용자 액션으로 생성 (불필요 호출 제거)
   - `max_tokens` 작업별 상한 고정, free 플랜 월 기록 한도(M3-5)가 AI 비용 상한 역할
6. **무료 티어 최대한 유지** — Vercel Pro 전환은 결제 시작(M3)에만, Supabase Pro는 필요해질 때.
7. **모니터링으로 검증 (L-8)** — 작업별 토큰 사용량 로깅, "유저당 월 AI 원가" 대시보드.
   목표 상한: **$0.10/유저/월** (기존 목표 $0.5에서 하향).

## 6. 리스크 / 메모

- 단가는 자주 바뀐다 — 추상화 레이어(1번)가 있으면 단가 변동이 코드 변경 없이 흡수됨.
- 품질 하한선: 역질문·태깅은 저가 모델로 충분하다는 가정. 카드/불릿은 제품 가치의 핵심이므로
  품질 미달이면 비용보다 품질 우선 (시나리오 C까지는 여전히 목표 내).
- Claude 모델은 배제가 아니라 **후보 중 하나** — 추상화 위에서 품질/단가로만 경쟁시킨다.
